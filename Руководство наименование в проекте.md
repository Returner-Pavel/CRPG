# Руководство по наименованию в C-RPG
Это руководство предназначено для понимания каких-либо переменных, функций и классов в C-RPG

---
##  Переменные
### Формат имени:
	(type)(Scope)_(nameVariable) = type
- **type** ∈ {`any`, `int`, `float`, `bool`, `str`, `list`,
`tuple`, `dict`} - Тип переменной
- **Scope** ∈ {`G`, `L`, `T`} - Область применения (`Global`, `Local`, `Temp`)
- **nameVariable** - заданное вами имя переменной
### Пример
```python
anyG_data = None  
# Переменная глобальна, может хранить **любой** тип данных (НЕ СОВЕТУЮ)

intG_playerId = 184917245  
# Глобальная переменная, хранит целое число — ID игрока

floatL_playerHealth = 100.0  
# Локальная (в этом файле) переменная, хранит здоровье игрока с плавающей точкой

boolG_isAlive = True  
# Глобальная булева переменная — жив ли игрок

strL_playerName = "Arthur"  
# Локальная строковая переменная — имя игрока

listG_inventory = [101, 202, 303]  
# Глобальная переменная, список предметов (ID предметов)

tupleL_position = (1.0, 2.5, -3.7)  
# Локальная кортеж-переменная, хранит позицию (x, y, z)

dictG_stats = {"strength": 10, "agility": 8, "intelligence": 12}  
# Глобальный словарь — характеристики игрока

for intT_counter in range(0, 10):  
    # Временная (temp) переменная-счётчик в цикле  
    print(intT_counter)
```
### Советы
- При написании названия пишите, что она хранит.
- В названиях первое слово с маленькой, а остальные с большой как в примерах.
- Не пишите сокращения в названиях.
- Избегаете использования типа `any` из-за непонимания того, что она хранит.
- Не меняйте тип в переменной которая уже определена в коде.
- При объявлении переменной сразу задайте значение, пример: объявляете переменную с `HP`, запишите туда сразу `0`.

## Функции
### Формат имени:
	def  {typeExit}{Scope}_{NameFuncion}() -> {typeExit}:
- **typeExit** ∈ {`any`, `none`, `int`, `float`, `bool`, `str`, `list`,
`tuple`, `dict`} - Возвращаемый тип  
- **Scope** ∈ {`G`, `L`} - Область применения (`Global`, `Local`)
- **NameFuncion** - заданное вами имя функции
### Пример
Встроеные методы Python:
```python
def noneL_DoSomething() -> None:
    # Локальная функция, ничего не возвращает
    print("Просто делаю что-то и ничего не возвращаю")

def intG_ComputeId() -> int:
    # Глобальная функция, возвращает целое число
    return 12345

def floatG_CalculateHealth() -> float:
    # Глобальная функция, возвращает число с плавающей точкой (например, здоровье)
    return 99.9

def boolG_IsActive() -> bool:
    # Глобальная функция, возвращает булево значение
    return True

def strL_FormatName() -> str:
    # Локальная функция, возвращает строку
    return "Игрок"
```
Методы с библиотекой `typing`:
```python
from typing import Any, List, Tuple, Dict

def anyG_GetSomething() -> Any:
    # Глобальная функция, возвращает что угодно
    return "может быть чем угодно"

def listG_GetInventory() -> List[int]:
    # Глобальная функция, возвращает список целых чисел (например, ID предметов)
    return [101, 202, 303]

def tupleG_GetPosition() -> Tuple[float, float, float]:
    # Глобальная функция, возвращает кортеж трёх координат (x, y, z)
    return (1.0, 2.5, -3.7)

def dictG_GetStats() -> Dict[str, int]:
    # Глобальная функция, возвращает словарь, где ключ — имя характеристики, значение — её число
    return {"strength": 10, "agility": 8, "intelligence": 12}
```
### Советы
- При написании названия пишите что она делает.
- В названиях все первые буквы слова большие как в примерах.
- Не пишите сокращения в названиях.
- Избегайте использования типа `any` из-за непонимания того, что она может вернуть.
- Не возвращайте типы данных, которые не соответствует этой функций.
- Рекомендую в функциях использовать  `"""Комменты о действиях функции при наведении"""`
  Пример:
  ```python
  def intG_MultimpicatorNumber(intT_multimpicator, intT_number) -> int:
	 """
	 Коммент о функции, при наведении он будет отображатся.
	1. `intT_multimpicator` - множитель числа
	2. `intT_number` - само число
	 """ 
    return intT_multimpicator * intT_number

  ```
